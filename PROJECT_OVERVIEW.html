📋Introduction

The Dog-Cat Classifier is a production-grade, full-stack web application that leverages Artificial Intelligence and Machine Learning to classify images as either dogs or cats. Built with modern technologies, this project demonstrates expertise in frontend development, backend API design, database management, and AI integration.

Project Goal: Create a scalable web application where users can upload images and receive instant AI-powered predictions with confidence scores.

🛠️Technologies Used

Frontend (Web User Interface)

Framework: React.js (v18+)
Language: JavaScript (ES6+)
Routing: React Router DOM
HTTP Client: Axios
UI Components: React Icons (Feather & Lucide)
Notifications: React Hot Toast
Styling: Custom CSS3
State Management: Context API
Build Tool: Webpack (via Create React App)

Frontend Features:

  • Responsive UI with drag-and-drop file upload
  • Real-time image preview before classification
  • Authentication with JWT tokens
  • User profile management
  • Result visualization with confidence bars
  • CSV download of results
  • Mobile-friendly design

Backend (Server & API)

Framework: FastAPI (Python)
Server: Uvicorn (ASGI)
Language: Python 3.8+
Database: MySQL / MariaDB
ORM: MySQL Connector Python
Authentication: JWT (JSON Web Tokens)
API Documentation: Swagger UI / OpenAPI
Password Security: PBKDF2 Hashing

Backend Features:

  • RESTful API endpoints
  • JWT-based authentication & authorization
  • File upload handling with validation
  • Secure password hashing (PBKDF2)
  • Email & phone number validation
  • CORS support for frontend communication
  • Comprehensive error handling & logging
  • Image serving endpoint

Deep Learning / AI Model

Framework: TensorFlow 2.13+ / Keras
Model Type: Convolutional Neural Network (CNN)
Input Size: 128 × 128 × 3 (RGB images)
Loss Function: Binary Crossentropy
Optimizer: Adam (learning_rate=0.001)
Metrics: Accuracy

Model Architecture:

Input Layer (128×128×3)
    ↓
Data Augmentation Layer
    ↓
Rescaling Layer (1/255 normalization)
    ↓
Conv2D (32 filters) + BatchNorm + MaxPool
    ↓
Conv2D (64 filters) + BatchNorm + MaxPool
    ↓
Conv2D (128 filters) + BatchNorm + MaxPool
    ↓
Conv2D (256 filters) + MaxPool
    ↓
Flatten
    ↓
Dense (256) + Dropout (0.5)
    ↓
Dense (1) + Sigmoid
    ↓
Output (0-1 probability)

Database

System: MySQL / MariaDB
Tables: users, images
Query Language: SQL

Database Schema:

  • users table: Stores user credentials, profiles, contact information
  • images table: Stores uploaded images, predictions, confidence scores, timestamps

Data Flow:

  1. User Authentication: User credentials → Validation → Password hashing → JWT token generation
  2. Image Upload: Select images → File validation → Server storage → Database entry
  3. Classification: Image upload → Preprocessing (128×128) → TensorFlow model → Prediction → API response
  4. Results Management: Display prediction → Store in database → Show history → Download CSV
